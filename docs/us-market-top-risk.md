# 美股顶部风险预判

## 模块边界

本模块用于定期计算美股顶部风险温度，不直接给出买卖指令。它独立于账号抓取和账号观点分析，后续可以作为单独页面展示，也可以写入数据库供前端读取。

当前研究脚本：

```bash
python3 research/market_top_risk/run_market_top_risk.py
```

当前研究输出：

- `research/market_top_risk/output/market_top_risk_report.md`
- `research/market_top_risk/output/indicator_value_near_high_summary.csv`
- `research/market_top_risk/output/weekly_market_top_risk.csv`
- `research/market_top_risk/output/latest_snapshot.json`

## v1 只保留的强指标

本模块 v1 只采用近高位回测里更强的一组指标。评估口径是：

> 当 Nasdaq 100 仍在 52 周高点 10% 以内时，信号出现后未来 26 周最大回撤是否比普通近高位周更糟。

近高位基准：

- 未来 26 周平均最大回撤：`-5.5%`
- 未来 26 周出现 10%+ 回撤概率：`21.1%`

强指标：

| 指标 | 模块 | 回测表现 | v1 用途 |
| --- | --- | --- | --- |
| `breadth_weakness_score` | 市场宽度 | 平均回撤 `-8.0%`，10%+ 回撤概率 `30.7%` | 核心预警 |
| `breakage_score` | 信用/金融条件组合 | 平均回撤 `-8.0%`，10%+ 回撤概率 `32.1%` | 核心确认 |
| `rsp_spy_13w_rel_pctl` | 市场宽度 | 平均回撤 `-7.9%`，10%+ 回撤概率 `28.2%` | 核心预警 |
| `nfci_13w_chg_pctl` | 金融条件 | 平均回撤 `-7.6%`，10%+ 回撤概率 `28.9%` | 核心确认 |
| `anfci_pctl` | 金融条件 | 平均回撤 `-7.4%`，10%+ 回撤概率 `27.7%` | 核心确认 |
| `qqew_qqq_13w_rel_pctl` | 市场宽度 | 平均回撤 `-7.3%`，10%+ 回撤概率 `25.9%` | 核心预警 |
| `credit_baa10y_pctl` | 信用 | 平均回撤 `-7.2%`，10%+ 回撤概率 `30.4%` | 核心确认 |

## 指标含义

### 市场宽度恶化

市场宽度指标不直接抓全市场成分股，而是用 ETF 相对表现做代理：

| 指标 | 计算 |
| --- | --- |
| `rsp_spy_13w_rel_pctl` | `RSP` 13 周收益减 `SPY` 13 周收益，再转历史分位 |
| `qqew_qqq_13w_rel_pctl` | `QQEW` 13 周收益减 `QQQ` 13 周收益，再转历史分位 |
| `breadth_weakness_score` | 上面两个宽度分位的弱化组合，分位越高表示宽度越差 |

解释：

- `RSP/SPY` 弱，表示等权 S&P 500 跑输市值加权 S&P 500，市场上涨依赖少数大市值股票。
- `QQEW/QQQ` 弱，表示等权 Nasdaq 100 跑输市值加权 Nasdaq 100，科技内部参与度变差。
- 这类信号在顶部预判里更接近前瞻预警。

### 金融条件转紧

金融条件使用 Chicago Fed 指数，通过 FRED 自动获取。

| 指标 | 计算 |
| --- | --- |
| `nfci_13w_chg_pctl` | `NFCI` 13 周变化值的历史分位 |
| `anfci_pctl` | `ANFCI` 绝对水平的历史分位 |

解释：

- `NFCI` 和 `ANFCI` 越高，金融条件越紧。
- `nfci_13w_chg_pctl` 捕捉金融条件从宽松转紧的速度。
- `anfci_pctl` 捕捉金融条件绝对压力。

### 信用利差

| 指标 | 计算 |
| --- | --- |
| `credit_baa10y_pctl` | FRED `BAA10Y` 的历史分位 |

解释：

- `BAA10Y` 是 Baa 公司债收益率相对 10 年期美债收益率的利差。
- 利差越高，信用风险越紧。
- 高收益 OAS 更贴近风险资产，但当前公开 FRED CSV 在本环境只覆盖较短历史，因此 v1 用 `BAA10Y` 做长期可回测代理。

## 数据来源

| 数据 | 来源 | 自动化方式 | 说明 |
| --- | --- | --- | --- |
| `NASDAQ100` | FRED | CSV 下载 | 主回撤标签和近高位过滤 |
| `NFCI` | FRED | CSV 下载 | 金融条件 |
| `ANFCI` | FRED | CSV 下载 | 调整后金融条件 |
| `BAA10Y` | FRED | CSV 下载 | 长期信用利差代理 |
| `SPY`、`RSP`、`QQQ`、`QQEW` | Nasdaq 公共历史接口 | JSON 下载 | 无 key，但当前只覆盖约 10 年 |

## 计算流程

1. 下载并缓存原始数据。
2. 将日频或周频数据统一到周五 as-of。
3. 计算 Nasdaq 100 是否仍在 52 周高点 10% 以内。
4. 计算各指标的 13 周相对收益、13 周变化或绝对水平。
5. 将指标转换为扩展历史分位数。
6. 在近高位样本中评估未来 26 周最大回撤。
7. 输出当前快照和指标价值排序。

## 风险状态解释

v1 页面建议只展示两个结论层：

| 层级 | 含义 |
| --- | --- |
| 宽度预警 | 市场仍在高位，但等权指数明显跑输市值加权指数 |
| 破裂确认 | 金融条件转紧或信用利差压力升高 |

解释方式：

- 只有宽度预警：市场结构变脆，但不代表顶部已经确认。
- 宽度预警 + 金融条件转紧：顶部风险显著升温。
- 宽度预警 + 信用利差高：风险从股票内部扩散到信用市场。
- 三者同时出现：进入高风险状态，适合展示为红色风险标签。

## 不纳入 v1 核心的指标

下列指标可以保留在研究表中，但不进入 v1 核心展示：

| 指标 | 原因 |
| --- | --- |
| `margin_debt_yoy_pctl` | 有背景价值，但单独不是好时点信号 |
| `equity_pc_20w_inv_pctl` | 回测有一定价值，但公开 Cboe 历史 CSV 当前只到 2019 |
| `vix_pctl` | 更像压力已经出现后的确认 |
| `IWM/SPY`、`IPO/SPY`、`ARKK/QQQ` | 当前回测不支持作为核心顶部信号 |
| `NDX below 40w`、`40w slope negative` | 更像已经破位后的状态 |
| `AI leader/QQQ` | 当前回测不支持单独作为顶部信号 |

## 调度建议

该模块不需要高频运行。

建议：

- 每 3 到 5 天运行一次。
- 输出最新快照和历史表。
- 后续接入页面时只读取最近一条快照和最近 1 年走势。

## 设计约束

- 不把单个指标包装成买卖点。
- 页面展示应明确区分“预警”和“确认”。
- 所有阈值用历史分位数，不写死绝对值。
- 数据源不可用时，指标应标记为缺失，不用旧值无限 forward-fill。
- 研究输出和运行缓存不应替代正式数据库设计。
